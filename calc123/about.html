Основная идея (кратко)

В админке вы создаёте калькулятор:

  • даёте ему название и формулу (в привычной математической записи);
  • перечисляете переменные (какие поля увидит пользователь на фронтенде — либо какие подставляются автоматически);
  • для каждого калькулятора можно указать символ валюты (или текст) и позицию (перед/после результата);
  • при необходимости включаете математическую капчу.

На странице сайта разместите шорткод [calc123 id="N"], где N — ID калькулятора из админки. Пользователь вводит значения переменных, нажимает «Рассчитать» и получает один числовой результат.

Правила записи переменных и формулы

  • Имена переменных содержат латинские буквы, цифры и подчёркивания, например a, price, num_items, x1.

    • Начинаться имя должно с буквы или подчёркивания: a, r1, _tmp.
    • Чувствительность к регистру: используйте одинаковый регистр в формуле и при создании переменных.

  • Формула задаётся в привычной математической форме и поддерживает:

    • операторы +, -, *, /, ^ (степень);
    • скобки ( ) для управления приоритетом;
    • сравнения >, <, >=, <=, ==, != (результат — 1 при true, 0 при false);
    • функции:
      • IF(cond, a, b) — возвращает a, если cond истинно (не 0), иначе b;
      • MAX(a, b), MIN(a, b) — максимум/минимум из двух аргументов;
      • ROUND(a, decimals) — округление a до заданного числа знаков.

  • Примеры корректных выражений:

    • a + b;
    • a + b * (c ^ 2);
    • IF(weight > 10, 100 + weight * 5, 50 + weight * 3);
    • ROUND(price * (1 - discount), 2).

Как задавать переменные в админке

При создании или редактировании калькулятора вы добавляете строки с переменными:

  • var_name[] — код переменной (например a, distance);
  • var_label[] — подпись, которую увидит пользователь (например «Длина, км», «Вес, кг»);
  • var_type[] — значение number или select;
  • var_options[] — для select строка вида Label1:Value1,Label2:Value2 (например Standard:1000,Premium:2000 — в формулу подставится выбранное значение);
  • var_hide[] — если отмечено, поле не показывается на фронтенде, но его значение участвует в расчёте (удобно для скрытых предустановок).

Несколько примеров

1) Простой пример: a + b

Задача: создаёте калькулятор со следующими настройками.

  • Формула: a + b.
  • Переменные:
    • a — метка «Число A», тип number;
    • b — метка «Число B», тип number.

Фронтенд: два обязательных поля. Например, если ввести A = 24 и B = 12, результат — 36 (плюс валюта, если задана).

2) Доставка: тарифы по дистанции и весу

Цель: если дистанция > 100 км — расчёт 1000 + weight * 50, иначе 500 + weight * 30.

Формула: IF(distance > 100, 1000 + weight * 50, 500 + weight * 30).

Переменные:

  • distance — «Дистанция (км)», тип number;
  • weight — «Вес (кг)», тип number.

Поведение:

  • distance = 120, weight = 10 → 1000 + 10 * 50 = 1500;
  • distance = 80, weight = 10 → 500 + 10 * 30 = 800.

3) Стоимость сайта с выбором пакета и скрытыми ценами

Задача: пользователь выбирает пакет (Standard/Premium) и количество страниц. Итог = цена пакета + pages * per_page.

  • Формула: package_price + pages * per_page.
  • Переменные:
    • package — «Пакет», тип select, опции Standard:10000,Premium:20000;
    • package_price — скрытая переменная; при желании можно вместо неё в формуле использовать сразу package и тогда формула станет package + pages * per_page;
    • pages — «Страниц», тип number;
    • per_page — «Цена за страницу», тип number (можно задать скрытую переменную со значением по умолчанию).

Как задавать select: используйте строку опций Standard:10000,Premium:20000 — на фронтенде выводятся метки, а в формулу подставляются числовые значения.

4) Формулы с округлением и степенью

  • Площадь круга (радиус r): 3.14159 * r ^ 2.
  • Округление до 2 знаков: ROUND(price * 0.87, 2).

5) Использование сравнений в арифметике

Сравнение возвращает 1 или 0. Например:

  • (a > b) * 100 вернёт 100, если a > b, иначе 0;
  • вложенные условия: IF(a > 100, IF(b > 50, a * 0.2, a * 0.15), a * 0.1).

Поведение фронтенда и валидация

  • Все видимые (не скрытые) поля отмечены required, поэтому их нельзя отправить пустыми.
  • Тип number ожидает числовое значение — проверка есть на клиенте и на сервере.
  • Если JavaScript отключён, сервер всё равно проверит данные и вернёт JSON-ошибку, которую плагин покажет пользователю.
  • Возможные сообщения об ошибках:
    • «Поле "X" обязательно для заполнения.» — пропущено обязательное поле;
    • «Поле "X" должно быть числом.» — введён текст вместо числа;
    • «Деление на ноль» — попытка делить на 0;
    • «Пустая формула», «Некорректный токен капчи», «Неправильный ответ капчи» и т. д.

Капча

  • Для каждого калькулятора можно включить простую математическую капчу (два числа 1–9, пользователь вводит сумму).
  • Капча проверяется на сервере и обновляется после каждого успешного расчёта.
  • Капча защищена токеном, чтобы нельзя было подделать числа на клиенте.

Валюта и отображение результата

  • Для каждого калькулятора можно указать символ валюты или текст (например ₽, «руб.», USD) и позицию (before или after).
  • Если поле валюты оставить пустым, результат выводится без неё.

Пример: currency_display = "₽", currency_pos = "after" → результат 123.45 ₽.

Шорткод и несколько калькуляторов

  • В админке можно создать несколько калькуляторов — каждый получит свой ID.
  • Шорткод выглядит так: [calc123 id="3"] — отобразит калькулятор с ID 3.
  • Если ID не указан, используется первый созданный калькулятор.

Пример строки с настройкой переменной

  • var_name[] = price;
  • var_label[] = Цена за штуку;
  • var_type[] = number;
  • var_options[] = (пусто);
  • var_hide[] = (отметить, если нужно скрыть).

Для select:

  • var_name[] = package;
  • var_label[] = Выберите пакет;
  • var_type[] = select;
  • var_options[] = Standard:10000,Premium:20000;
  • var_hide[] = (обычно не ставят — пользователь должен выбрать значение).

Небольшие советы

  1. Имена переменных в формуле должны в точности совпадать с var_name (регистр и символы).
  2. Не используйте русские буквы в именах переменных — только латиница, цифры и подчёркивание.
  3. Если нужна константа π, используйте значение 3.14159 или создайте скрытую переменную pi = 3.14159.
  4. Для тарифов удобнее использовать select с парой Label:Value — пользователь видит понятные названия, а в формулу подставляется число.
  5. Для сложных условий комбинируйте сравнения и вложенные IF.
  6. Тестируйте формулу в админке: вводите простые значения и проверяйте ожидаемый результат.

Дополнительные примеры

Пример A — калькулятор доставки.

  • Формула: IF(distance > 100, 1000 + weight * 50, 500 + weight * 30).
  • Ввод: distance = 120, weight = 5 → результат 1250.
  • Ввод: distance = 80, weight = 5 → результат 650.

Пример B — расчёт стоимости сайта.

  • Формула: package + pages * per_page.
  • Опции select: Standard:10000,Premium:20000.
  • per_page = 1500.
  • Пример: pages = 5, выбран Premium → 20000 + 5 * 1500 = 27500.

Пример C — скидка и округление.

  • Формула: ROUND(price * (1 - discount / 100), 2).
  • price = 1234.567, discount = 10 → ROUND(1234.567 * 0.9, 2) = ROUND(1111.1103, 2) = 1111.11.
